# Nikel — Obsidian плагин для работы с локальной LLM и построения графа знаний

**Nikel** — плагин для Obsidian, который:
- Обрабатывает текст через локальную LLM (Ollama) прямо из редактора
- Строит **граф знаний** из PDF-документов: извлекает материалы, эксперименты, свойства, процессы, публикации и связи между ними
- Отвечает на вопросы на основе построенной базы знаний

---

## Что вы получили

1. **Плагин** — папка `nikel-plugin/` (исходники + `main.js`)
2. **Источники** — папка `Источники информации/` с подпапками:
   - `Статьи/` — научные статьи
   - `Доклады/` — технические доклады
   - `Журналы/` — журнальные публикации
   - `Материалы конференций/` — труды конференций
   - `Обзоры/` — обзорные статьи

## Шаг 1. Установка плагина в Obsidian

1. Скопируйте папку `nikel-plugin/` в `<хранилище>/.obsidian/plugins/nikel-plugin/`
2. Откройте Obsidian → **Настройки** → **Сторонние плагины** → включите **Включить загрузку из сторонних источников**
3. Нажмите **Перезагрузить** (иконка обновления) → в списке появится **Nikel**
4. Включите его тумблером

## Шаг 2. Настройка

1. Откройте **Настройки** → **Nikel**
2. **Ollama URL** — оставьте `http://localhost:11434` (если Ollama запущен локально)
3. **Модель** — `gemma4:e4b` (или любая модель с поддержкой Vision)
4. **Папка с PDF** — нажмите **«Обзор»** и выберите папку `Источники информации` (или введите путь вручную)

## Шаг 3. Запуск Ollama

Если Ollama ещё не установлен:

```bash
# Установка
curl -fsSL https://ollama.com/install.sh | sh

# Запуск
ollama serve

# В другом терминале — загрузка модели
ollama pull gemma4:e4b
```

## Шаг 4. Первая индексация

1. В Obsidian нажмите `Cmd+P` (Ctrl+P) → выберите **Nikel: Индексировать PDF-папку**
2. Появится окно прогресса — плагин:
   - Сканирует все PDF в `Источники информации/` и подпапках
   - Для каждого PDF: рендерит страницы → Vision LLM → извлекает сущности и связи
   - Сохраняет граф знаний в `nikel/index.json`
   - Генерирует `.md`-файлы для каждой сущности (в `nikel/materials/`, `nikel/publications/` и т.д.)
   - Создаёт `nikel/_index.md`, `nikel/_graph.md` (Mermaid), `nikel/canvas/обзор-базы-знаний.canvas`
3. Ждите завершения. Первая индексация — самая долгая.

## Шаг 5. Как пользоваться

### Запрос к базе знаний (с графом)

1. В любом документе Obsidian напишите `@nikel_s` на новой строке
2. Ниже напишите вопрос, например:
   ```
   @nikel_s
   Какие методы обессоливания воды подходят для сульфатов до 300 мг/л?
   ```
3. Нажмите `Cmd+P` → **Process @nikel task** (или назначьте горячую клавишу)
4. Плагин найдёт релевантные сущности в графе и ответит со ссылками на источники
5. Ответ сохранится в `nikel/_answers/` как отдельный документ

### Обычный запрос к LLM (без графа)

- `@nikel_s` работает и как обычный AI-ассистент, если граф ещё не построен
- `@nikel_f` — исправление форматирования и грамматики

### Добавление новой команды

1. Настройки → Nikel → **Добавить команду**
2. Заполните триггер (`@nikel_t`), описание, промпт
3. Шаблон `{{input}}` заменяется на текст пользователя

## Шаг 6. Просмотр базы знаний

- **`nikel/_index.md`** — каталог всех сущностей по типам (12 типов)
- **`nikel/_graph.md`** — Mermaid-диаграмма связей (в Obsidian отрендерится как граф)
- **`nikel/canvas/обзор-базы-знаний.canvas`** — интерактивный граф в Obsidian Canvas
- Подпапки `nikel/materials/`, `nikel/publications/`, `nikel/processes/` и т.д. — каждая сущность = заметка с Dataview-полями и ссылками

## Шаг 7. Переиндексация

При добавлении новых PDF снова запустите **Nikel: Индексировать PDF-папку**.
Плагин сравнивает MD5-хеши и обрабатывает только новые и изменённые файлы. Существующие заметки не перезаписываются.

## Требования к системе

- **Ollama** (локально или на сервере)
- **Модель с Vision** (gemma4:e4b, llava, bakllava и т.п.)
- **Obsidian** v1.5+
- macOS / Windows / Linux
- ~1.1 МБ на плагин + место под сгенерированные заметки

| Тип сущности | Описание |
|---|---|
| `material` | Материалы, сплавы, составы, вещества |
| `experiment` | Эксперименты, испытания, протоколы |
| `property` | Свойства, характеристики, концентрации |
| `mode` | Режимы, параметры процессов |
| `equipment` | Оборудование, установки, приборы |
| `team` | Лаборатории, организации |
| `person` | Исследователи, авторы |
| `conclusion` | Выводы, результаты |
| `topic` | Темы, области исследований |
| `publication` | Статьи, патенты, отчёты, обзоры |
| `process` | Технологические процессы (выщелачивание, плавка...) |

| Тип связи | Описание |
|---|---|
| `uses_material` | Процесс/эксперимент → материал |

| `in_mode` | Эксперимент/процесс → режим |
| `uses_equipment` | Процесс/эксперимент → оборудование |
| `conducted_by` | Эксперимент → команда/персона |
| `leads_to` | Режим/эксперимент → вывод |
| `described_in` | Сущность → публикация (источник) |
| `operates_at_condition` | Процесс → условие работы |
| `produces_output` | Процесс → материал/вывод |
| `validated_by` | Вывод → эксперимент/публикация |
| `contradicts` | Вывод противоречит выводу |
| `related_to` | Общая связь |
| `precedes` | Хронология |

## Устранение неполадок

### Ошибка подключения к Ollama

Плагин автоматически пробует `127.0.0.1` если `localhost` не отвечает. Если ошибка остаётся:
1. Убедитесь, что Ollama запущен: `ollama list`
2. Проверьте брандмауэр (порт 11434)

### Пустой ответ от модели
- Проверьте выбранную модель в настройках
- Загрузите модель: `ollama pull gemma4:e4b`

### «Не найден @nikel_* триггер»
- Триггер должен быть **выше** курсора на отдельной строке
