# Nikel — Obsidian плагин для работы с локальной LLM (Ollama)

**Nikel** — плагин для Obsidian, который позволяет обрабатывать текст с помощью локальной LLM-модели (Ollama) прямо из редактора заметок.

## Для чего

- Вы пишете в заметке `@nikel_s <задача>` — модель выполняет задачу
- Вы пишете `@nikel_f <текст>` — модель исправляет форматирование
- Плагин поддерживает любое количество пользовательских команд с собственными промптами
- Всё работает локально, данные не уходят в интернет

## Требования

- **Obsidian** v1.5.0 или выше
- **Ollama** установлен и запущен (`ollama serve`)
- Модель `gemma4:e4b` (или другая, можно выбрать в настройках)

## Установка

```bash
# 1. Скопировать папку плагина в vault
cp -r nikel-plugin ~/path/to/vault/.obsidian/plugins/nikel

# 2. Перезапустить Obsidian
# 3. Settings → Community plugins → Nikel → Enable
```

## Настройка

Откройте **Settings → Nikel**:

| Параметр | Описание |
|----------|----------|
| **Ollama URL** | Адрес сервера (по умолч. `http://localhost:11434`) |
| **Модель** | Выбор модели из списка установленных |
| **Команды** | Список команд `@nikel_*` с триггером, описанием и промптом |

### Команды по умолчанию

| Триггер | Описание |
|---------|----------|
| `@nikel_s` | Отправить задачу модели (универсальный промпт) |
| `@nikel_f` | Исправить форматирование и грамматику |

## Использование

1. Откройте заметку в Obsidian
2. Начните вводить `@nik` — появится автокомплит с доступными командами
3. Выберите команду (например `@nikel_s`) и напишите задачу:
   ```
   @nikel_s Объясни простыми словами что такое Dependency Injection
   ```
4. Нажмите `Cmd+P` → **Process @nikel task** (или назначьте горячую клавишу)
5. Плагин найдёт триггер выше курсора, отправит задачу в Ollama и вставит ответ под строкой

Результат:
```
@nikel_s Объясни простыми словами что такое Dependency Injection

> **Nikel (gemma4:e4b):**
> Dependency Injection — это паттерн, при котором объект получает свои зависимости извне...
>
```

## Добавление новой команды

1. Settings → Nikel → **Добавить команду**
2. Заполните:
   - **Триггер**: `@nikel_t`
   - **Описание**: `Перевести текст на английский`
   - **Промпт**: `Переведи следующий текст на английский. Отвечай только переводом, без пояснений:\n\n{{input}}`
3. Сохраните

Теперь при вводе `@nik` в редакторе появится новая команда `@nikel_t`.

## Устранение неполадок

### Windows: ошибка "Не удалось выполнить запрос к Ollama"

Плагин автоматически пробует `127.0.0.1` если `localhost` не отвечает (IPv6-проблема).
Если ошибка остаётся:

1. Убедитесь, что Ollama запущен: откройте `cmd` и выполните `ollama list`
2. Проверьте, что брандмауэр Windows не блокирует порт 11434
3. Проверьте, что в Obsidian Settings → Community plugins → Nikel плагин включён

### Пустой ответ от модели

- Убедитесь, что в настройках выбрана правильная модель
- Проверьте, что модель загружена: `ollama list`
- Если модель не загружена: `ollama pull gemma4:e4b`

### "Не найден @nikel_* триггер"

- Убедитесь, что триггер (`@nikel_s`) находится **выше** курсора
- Триггер должен быть на отдельной строке (или в начале строки)
- Проверьте, что команда включена в Settings → Nikel

## Knowledge Graph (в разработке)

Плагин умеет строить базу знаний из PDF-документов:

1. Укажите папку с PDF в настройках
2. Нажмите **Индексировать**
3. Плагин сам:
   - Распарсит PDF через мультимодальную LLM (gemma4:e4b)
   - Извлечёт сущности: материалы, эксперименты, свойства, режимы, установки, команды
   - Построит граф связей между ними
   - Сгенерирует `.md` документы с `[[links]]`, `#tags`, Dataview-полями
   - Сгенерирует `.canvas` для Obsidian Canvas
4. Вводите `@nikel_s что делали по сплаву X при режиме Y?` — ответ придёт из базы знаний

Подробнее: [PLAN.md](PLAN.md)
